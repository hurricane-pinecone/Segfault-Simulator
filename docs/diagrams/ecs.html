Systems

A System declares required components via registerComponent<T>(), keeps its own entities list, and runs update(dt) / render(). Disabled systems are skipped.

Frame loop

Game::run → SceneManager::current() → Scene
1 · processInput
Scene::processInput → GameObjects' onProcessInput (poll input, set velocities).
→
2 · update(dt)
Scene::update: flushEntities() applies deferred add/remove & re-matches systems → every enabled system.update(dt) → GameObjects' onUpdate.
→
3 · render
Scene::render: every enabled system.render() submits quads to the renderer → SDL_GL_SwapWindow.
↻ repeat — structural changes (create/destroy) are queued and applied at the top of the next update, so a frame iterates a stable set of entities.
