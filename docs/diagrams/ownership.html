The OOP layer

GameObject

A GameObject is a thin OOP wrapper that owns an Entity handle into the scene's Registry — it does not store component data itself. Scene::createObject<T>() constructs it and calls onCreate(scene), where it spawns its entity and attaches components.

GameObject  (base)
Entity m_entity;  ·  hooks: onCreate / onUpdate / onProcessInput.
Composes behaviour over the ECS — input & per-object logic live here, simulation lives in Systems.
